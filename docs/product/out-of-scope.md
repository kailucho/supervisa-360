# Fuera del alcance del MVP — Supervisa 360

Este documento lista explícitamente lo que el MVP **no** incluirá, el motivo de la
exclusión y si se considera una fase futura posible.

| Función excluida                                             | Motivo de exclusión                                                                                                                                                                                                            | ¿Fase futura?                                                                    |
| ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------- |
| Usuario de jefatura                                          | El MVP atiende únicamente a las 2 supervisoras; jefatura no tiene un flujo definido aún (qué ve, qué permisos tiene).                                                                                                          | Sí — probable v2, una vez validado el uso por las supervisoras.                  |
| Acceso para asesores                                         | Los asesores no participan como usuarios de la app; solo son referenciados como dato de la asociación/visita.                                                                                                                  | No previsto en el corto plazo; requeriría definir qué verían y con qué permisos. |
| Selección formal de asesores por mes                         | No se definió un proceso de asignación mensual; solo se registra el asesor actual de cada asociación.                                                                                                                          | Posible, si ADRA formaliza ese proceso.                                          |
| Retroalimentación al asesor dentro de la app                 | No hay usuario asesor ni flujo de comunicación definido.                                                                                                                                                                       | Depende de si se habilita acceso de asesores.                                    |
| Registro de acta de compromiso                               | No se especificó su contenido ni su flujo; la evaluación detallada vive en la herramienta oficial de ADRA.                                                                                                                     | Posible, si se define su estructura.                                             |
| Carga de fotografías o archivos                              | No se definió almacenamiento de archivos ni su necesidad para el MVP.                                                                                                                                                          | Sí, es una extensión natural del registro de visita.                             |
| Funcionamiento sin conexión                                  | Las supervisoras usan la app en contexto con conexión disponible; añade complejidad de sincronización.                                                                                                                         | Baja prioridad; solo si se identifica una necesidad real de campo sin señal.     |
| Aplicación móvil nativa                                      | El MVP es una web app (React/Vite); una app nativa duplicaría esfuerzo sin necesidad comprobada.                                                                                                                               | Posible, si el uso desde el móvil vía navegador resulta insuficiente.            |
| Calendario gráfico (semanal/mensual)                         | El MVP usa una lista ordenada por fecha, más simple de construir y suficiente para 2 usuarias.                                                                                                                                 | Sí, mejora de UI razonable a futuro.                                             |
| Importación de Excel desde la interfaz                       | La carga inicial se hace por CSV directo a la base de datos; una pantalla de importación es esfuerzo adicional no crítico para el MVP.                                                                                         | Sí, si el volumen de actualizaciones lo justifica.                               |
| Notificaciones                                               | No se definieron canales (email, push, etc.) ni eventos disparadores.                                                                                                                                                          | Sí, útil para avisos de duplicidad o metas.                                      |
| Recordatorios                                                | Mismo motivo que notificaciones.                                                                                                                                                                                               | Sí.                                                                              |
| Exportación a Excel o PDF                                    | No es necesaria para la coordinación diaria entre 2 supervisoras; se prioriza la consulta en pantalla.                                                                                                                         | Sí, si se requiere reportar a jefatura.                                          |
| Gráficos de evolución                                        | El MVP deja la interpretación de mejora/estancamiento al criterio de la supervisora, revisando el historial manualmente.                                                                                                       | Sí, natural una vez que haya suficiente histórico.                               |
| Tendencia automática (mejora/empeora)                        | Requiere definir una lógica de cálculo que no está especificada; se prefiere no inventarla en el MVP.                                                                                                                          | Sí, junto con los gráficos de evolución.                                         |
| Recomendación automática de asociaciones a visitar           | No se definieron criterios de priorización.                                                                                                                                                                                    | Posible, como mejora de planificación.                                           |
| Integración con la herramienta oficial de ADRA               | Fuera del alcance por ahora; esta app coexiste con ella sin reemplazarla.                                                                                                                                                      | Posible a largo plazo, si ADRA lo requiere.                                      |
| Ubicación GPS                                                | No se identificó una necesidad de negocio para el MVP.                                                                                                                                                                         | Baja prioridad.                                                                  |
| Firma digital                                                | No se definió su propósito ni validez legal requerida.                                                                                                                                                                         | Baja prioridad.                                                                  |
| Evaluación por aspectos / puntuación por aspecto             | Es responsabilidad de la herramienta oficial de ADRA; este MVP solo registra una puntuación y un comentario general.                                                                                                           | No previsto; duplicaría la herramienta oficial.                                  |
| Creación o eliminación de asociaciones/asesores desde la app | La creación/baja de asociaciones y asesores, y la edición de sus datos de identidad (código, nombre, región), se gestionan por reimportación de CSV en el MVP; solo el estado y el asesor asignado son editables desde la app. | Posible, si se decide dar de baja el flujo de CSV a favor de un CRUD completo.   |
